# 📑 S08: Dynamic Ports Specification (Port 0)

**Status:** 📝 Draft  
**Owner:** Engineering Team

## 1. Goal
Provide support for **Dynamic Port Binding** (Port 0) to allow the Operating System to assign an available port in real-time. This is critical for **Automated Testing (CI)** and **High-Fidelity Demos**, preventing "Socket address already in use" errors when multiple instances or tests run concurrently.

## 2. Requirements

### 2.1. Port Retrieval
The framework must provide a mechanism to retrieve the actual port assigned by the OS after the server starts. Port 0 is only a signal for dynamic assignment; once bound, the server must know its active port.

### 2.2. Interface Evolution
- **IWebHost**: Add a `Port` property or `GetPort` method.
- **IWebApplication**: Add a `Port` property to mirror the active host's port.

### 2.3. Server Adapters Support
- **Indy Adapter**: Must handle `DefaultPort := 0` and extract the assigned port from `Bindings`.
- **DCS Adapter**: Must handle port 0 and extract the assigned port from the underlying IOCP/Epoll engine.

### 2.4. Logging & Diagnostics
The startup log must reflect the **Real Port** instead of the configured port 0.
- *Current*: `Dext server running on http://localhost:0`
- *Expected*: `Dext server running on http://localhost:54321`

## 3. Technical Implementation Details
